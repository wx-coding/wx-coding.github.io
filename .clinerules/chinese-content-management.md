## Brief overview
This file contains guidelines for managing Chinese content in the wx-coding Hugo blog, focusing on proper Chinese typography, content structure, and localization best practices.

## Chinese typography standards
- **Punctuation**: Use full-width Chinese punctuation（，。！？；：""''（）【】）
- **Spacing**: No spaces between Chinese characters, add spaces around Latin letters and numbers
- **Line breaks**: Avoid breaking Chinese words across lines, use proper word segmentation
- **Quotation marks**: Use Chinese quotation marks "" for primary quotes, '' for nested quotes

## Content structure guidelines
- **Title format**: Use clear, descriptive Chinese titles that accurately reflect content
- **Headings**: Maintain consistent Chinese heading hierarchy with proper numbering
- **Paragraphs**: Keep paragraphs concise, 3-5 sentences recommended for readability
- **Lists**: Use Chinese numbering （一、二、三、） or bullet points with consistent indentation

## Technical content in Chinese
- **Code terminology**: Maintain English for code keywords, function names, variable names
- **Technical terms**: Provide Chinese translations with English originals in parentheses when first introduced
- **Code comments**: Use Chinese comments in code examples when explaining to Chinese audience
- **Command line**: Keep commands in English, provide Chinese explanations

## Markdown formatting for Chinese
- **Emphasis**: Use `**粗体**` and `*斜体*` appropriately for Chinese text emphasis
- **Code blocks**: Ensure Chinese characters in code comments are properly encoded
- **Links**: Use descriptive Chinese text for link anchors
- **Images**: Provide Chinese alt text for accessibility

## SEO considerations for Chinese content
- **Keywords**: Use relevant Chinese technical keywords naturally throughout content
- **Meta descriptions**: Write compelling Chinese descriptions under 160 characters
- **URL slugs**: Use pinyin or English for file names, Chinese for display titles
- **Tags**: Use Chinese tags that reflect content categories and search behavior

## Content quality standards
- **Accuracy**: Ensure technical translations are accurate and contextually appropriate
- **Readability**: Use clear, professional Chinese suitable for technical audience
- **Consistency**: Maintain consistent terminology throughout the article series
- **Citations**: Properly attribute sources using Chinese citation standards

## Localization best practices
- **Cultural context**: Consider Chinese cultural context in examples and analogies
- **Date formats**: Use Chinese date formats like "2025年1月12日"
- **Numbers**: Use Arabic numerals with Chinese units（如：100个、50%）
- **Units**: Use standard Chinese units and measurement systems

## Content workflow
- **Draft review**: Have native Chinese speakers review technical translations
- **Proofreading**: Check for character encoding issues and display problems
- **Testing**: Verify content displays correctly across different browsers and devices
- **Updates**: Keep Chinese content synchronized with any English versions

## Tools and resources
- **Input methods**: Recommend proper Chinese input methods for accurate typing
- **Dictionary**: Use reliable Chinese-English technical dictionaries
- **Grammar check**: Utilize Chinese grammar checking tools
- **Encoding**: Ensure UTF-8 encoding for all files containing Chinese text
